# FigJam Diagram Legend: Red, Blurry & Remediate Areas

**Open your FigJam diagrams** using the claim links from the diagram widget. Each diagram is in a separate FigJam file.

---

## Diagram 1: CURRENT STATE

### RED (Critical Gaps)
| Node | Explanation |
|------|-------------|
| **No faceted sort** | Sort applies globally, not within filtered results. User filters to "Roll-On" then sorts by price—cheap accessories can appear before bottles. Baymard: only 10% of sites fix this. |
| **No thread size on bottle cards** | Thread size (18-415, 20-410) is critical for fitment. Users must click into PDP to see it. Baymard: category-specific attributes should appear in list view. |
| **No color swatches in grid** | Variant count shown but not which colors. Users can't compare at a glance. |

### BLURRY (Confusion)
| Node | Explanation |
|------|-------------|
| **6 entry paths - overwhelming** | Hero CTA, PathChooser (3), Start Here (6 cards), Design Families (9), Navbar. New users don't know where to start. |
| **PathChooser equal weight** | Search, Guided, and Grace have same visual prominence. Power users want search first; uncertain users want Guided. |
| **Guided flow hidden** | "Help Me Choose" is one of three equal options. Many users never discover the 3-step wizard. |

### REMEDIATE
| Node | Action |
|------|--------|
| Add faceted sorting | Sort within filtered set only. |
| Category-specific card attributes | Thread size on bottles, applicator on components. |
| Sticky filter summary bar | Show active filters + count when scrolled; easy "Clear all." |

## Diagram 2: OPTION A (Catalog-First)

### RED (Remaining Gaps)
| Node | Explanation |
|------|-------------|
| **Guided flow less prominent** | Removed PathChooser; Guided becomes overlay in catalog. Users who prefer "help me choose" may miss it. |
| **Power users may want direct family browse** | Single search + catalog. Design Families carousel removed. Users who browse by aesthetic lose that path. |

### BLURRY
| Node | Explanation |
|------|-------------|
| **Single path - some users prefer visual browse** | Search-first assumes users know terms. Visual browsers (scroll, click images) get less support. |

### REMEDIATE
| Node | Action |
|------|--------|
| Add Browse by Family link | Restore design family quick access in catalog header. |
| Surface Guided in catalog header | "Not sure? Take the 3-step guide" prominent in catalog. |

## Diagram 3: OPTION B (Intent-First)

### RED (Critical)
| Node | Explanation |
|------|-------------|
| **Extra step friction for power users** | Qualifier "What are you packaging?" adds a click before catalog. Users who know exactly what they need feel slowed down. |
| **Qualifier can feel like gatekeeping** | Mandatory question before browse can frustrate. "Just let me look" is a valid intent. |

### BLURRY
| Node | Explanation |
|------|-------------|
| **Skip vs answer - unclear value** | If user skips qualifier, what do they get? Same catalog? Different default? Unclear. |
| **Quick Match vs full catalog - when to use** | "Top 12 for your use case" vs "View all" — when to show which? Can confuse. |

### REMEDIATE
| Node | Action |
|------|--------|
| Make qualifier optional and dismissible | "Skip — browse all" always visible. Pre-apply filters only if user answers. |
| Clear Skip - Browse all link | One-tap escape from qualifier. |

## Diagram 4: OPTION C (Recommended Hybrid)

### STRENGTHS (Green)
| Node | Explanation |
|------|-------------|
| **Multiple paths - no friction** | Search, Guided, Grace, Browse All all available. No mandatory extra steps. |
| **Baymard-aligned catalog** | Faceted sort, category-specific attributes, sticky filters. |
| **Power + new user friendly** | Power users get prominent search; uncertain users get Guided and Grace. |

### Minor Polish (Remediate)
| Node | Action |
|------|--------|
| Mobile filter drawer UX | Active filter count in drawer header; "View X results" to close. |

## How to Use in FigJam

1. **Claim each diagram** — Click the claim link for each diagram to open it in FigJam.
2. **Add sticky notes** — Use FigJam sticky notes to add the explanations above to each RED, BLURRY, REMEDIATE node.
3. **Combine into one board** — Copy all four diagrams onto a single FigJam board for side-by-side comparison.
4. **Annotate** — Use red highlights for RED nodes, yellow for BLURRY, green for REMEDIATE/strengths.
